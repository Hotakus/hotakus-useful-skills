# 像素美术设计约束

像素 2D 游戏中角色、场景、碰撞箱的美术设计准则。适用于 top-down / 俯视角 ARPG。

## 1. 角色像素尺寸

| 尺寸 | 优点 | 缺点 | 适用 |
|------|------|------|------|
| 16×16 | 工作量低，密集场景 | 战斗可读性差，动画极度受限 | 解谜、回合制 |
| 32×32 | 战斗清晰，动画够用 | 工作量中等 | **俯视 ARPG（推荐）** |
| 48×48 | 细节丰富，大招表现力强 | 工作量大，场景物件需等比放大 | 大型演出型 |

> 640×360 基础分辨率下，32px 高角色在屏幕上水平容纳约 18 个——ARPG 战斗的理想密度。

## 2. 角色体型与碰撞

俯视 ARPG 推荐 **32px 高 × 16px 宽**（横向体型），4 向动画。

碰撞箱只覆盖脚部（下半身 12-16px 区域），避免美术体型的视觉错位感。Godot 中用 `CollisionShape2D` 矩形直接拉到腿部。

```
正例：碰撞箱 = 角色脚底 16×12 矩形
反例：碰撞箱 = 角色全身 16×32 矩形 → 头顶吃伤害，观感怪异
```

## 3. 场景密度匹配

角色尺寸决定场景物件基准：

| 角色高 | 推荐 tile 尺寸 | 一棵树 | 一扇门 |
|-------:|:------------:|:-----:|:-----:|
| 32px | 16×16 | 48-64px | 32×48px |
| 48px | 16×16 或 32×32 | 64-96px | 48×64px |

> 原则：一个角色占 2×2 个 tile（32px 角色 + 16px tile）是最成熟的搭配。

## 4. TileMap 道路自然度

生硬不由 tile 大小决定，由**过渡 tile 是否充足**决定。

### 三层铺设

| 层 | 内容 | 作用 |
|----|------|------|
| 底层 | 道路 + autotiling 过渡 | 路→草地自动匹配 |
| 中层 | 碎石、杂草、裂缝 | 打破网格感 |
| 顶层 | 树根、路牌、断木 | 掩盖重复 |

### Godot 做法

1. TileSet 用 **Terrains** 定义"路"和"草"两种地形，设过渡规则
2. TileMapLayer 开 autotiling → 画路自动匹配边缘 tile
3. 画完后在中层手动点缀不规则装饰

> 主路用 L 形折线 + 少量弧线，不画纯对角线（autotiling 下易出奇怪对角）

## 5. 自检清单

- [ ] 角色尺寸匹配场景密度（32px 角色 + 16px tile）
- [ ] 碰撞箱仅覆盖脚部
- [ ] tileset 包含完整过渡 tile（路→草 4 向边缘）
- [ ] 中层有不规则装饰打破重复

## 6. 颜色策略

### 明度优先

明度对比 >> 色相对比。将截图转灰度检查——角色和背景之间差 3-4 个明度阶梯才算安全。

### 色相偏移

阴影不简单加黑。阴影往冷色偏（蓝紫），高光往暖色偏（黄白）。每个颜色都需要色相偏移，否则画面像蒙了一层灰。

### 饱和度分级

| 层级 | 饱和度 | 作用 |
|------|:-----:|------|
| 前景（角色/交互物） | 高 | 吸引视线 |
| 中景（地面/墙壁） | 中 | 提供语境 |
| 远景（天空/远山） | 低 | 退到背景 |

### 调色板规模

| 对象 | 颜色数 |
|------|:-----:|
| 32px 角色 | 6-12 |
| 16px 图块 | 4-8 |
| UI 图标 | 2-4 |
| 全游戏共用色板 | 16-64 |

> 加新颜色前，先试试把已有颜色做色相偏移。

## 7. 常见像素画陷阱

### 视觉噪声

```
反例：单像素飞点、零散抖动、孤立的 1×1 色点
正例：每个像素必须属于 ≥2×2 的色块（像素簇原则）
```
每几分钟缩到 1 倍大小看——认不出就砍细节。

### 32px 画太满

```
反例：在 32×32 上画铠甲纹理、腰带扣、披风褶皱 → 缩小后一坨色块
正例：5-6 种颜色，靠色块轮廓区分身体部位，2-2.5 头身
```
四肢不薄于 2px。

### 枕头着色

```
反例：边缘暗→中间亮，不管光源方向 → 物体像胀气的包子
正例：固定左上光源，左侧亮→右侧暗，顶面亮→底面暗
```

### 骷髅视觉

画面很美但分不清敌人/背景/交互物 = 没法玩。

```
正例：交互物与背景色相+明度双对比；敌人用动态闪烁辅助标识；道具用对比色外发光
```

## 8. UI/HUD 设计

### 字体

像素字体必须用原生像素尺寸的整数倍（8/16/24px），非整数 = 模糊。不用于长文本，只用于标题、按钮、HUD 标签。

### 图标

16×16 图标 2-4 色。缩到 8×8 能认出就是好图标。

### HUD 颜色避让

```
反例：主角是红色，血量条也是红色 → 战斗中混成一团
正例：HUD 用蓝/白，或完全不同饱和度的红
```

## 9. 动画帧数（32px 4 向）

| 动画 | 帧数 | 备注 |
|------|:---:|------|
| 待机呼吸 | 2-4 | 300-500ms/帧 |
| 行走 | 4-6 | 变量帧时长比加帧重要 |
| 攻击 | 2-4 | 蓄力快、命中保持、收招中等 |

### 制作技巧

- 只画三个方向（下、左、上），右 = 左翻转
- 上下走摆动 ≤2px，否则像滑冰
- 左右走手脚反相（左手前→右脚前）
- 头发/披风延迟 1 帧跟随身体
- 下方向上露出头顶更多，上方向上露出头发上缘更多

### Squash & Stretch

32px 上也有效——就 1-2px 的变化传达重量感：
```
跳跃蓄力：高 14→13px，宽 +1px
跳跃起飞：高 14→15px，宽 -1px
落地撞击：再压到 13px
```

## 10. Godot 像素陷阱

### Popup/Tooltip 模糊

Godot 4.4 中 Popup 会无视全局 Nearest 设置，使用 Bilinear 过滤。

修复：在 Autoload 中监听 `node_added`，检测 PopupPanel，手动设 `texture_filter = Nearest`。

### 资源导入

- 永远 PNG，不用 JPG
- 导入预设选 "2D Pixel"
- Filter 设 Point（Nearest）
- 缩放永远整数倍，禁止 "fit to container"

## 11. 自检清单

- [ ] 灰度图下角色和背景差 ≥3 明度阶梯
- [ ] 前景饱和 > 中景 > 远景
- [ ] 无孤点（每个像素属于 ≥2×2 色块）
- [ ] 无枕头着色（有固定光源方向）
- [ ] 战时可瞬间分辨敌人/背景/交互物
- [ ] 字体为原生像素尺寸整数倍
- [ ] 攻击动画蓄力快、命中保持
- [ ] 行走帧时长不均等分配
